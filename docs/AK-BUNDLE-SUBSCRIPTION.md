# AK推荐 - 批量订阅功能

## 功能概述

"AK推荐"是一个特殊的打包订阅选项，将92个HN热门技术博客打包成一个订阅源，用户只需一键添加，即可订阅所有优质技术博客。

## 功能特点

- ✅ **一键订阅92个博客** - 无需逐个添加
- ✅ **统一管理** - 所有博客源集中在一个订阅下
- ✅ **简洁界面** - 预设下拉菜单只显示4个选项（AK推荐、HackerNews、TechCrunch、Wired）
- ✅ **智能关键词过滤** - 预设关键词：tech, blog, programming, ai, startup
- ✅ **批量抓取** - 一次性抓取所有博客的最新内容
- ✅ **灵活控制** - 可以单独启用/禁用某个博客源

## 使用方法

### 1. 添加AK推荐订阅

1. 打开"订阅管理"对话框
2. 在预设公司下拉菜单中选择"**AK推荐**"（列表第一项）
3. 选择抓取频率（每小时/每天/每周）
4. 点击"添加"按钮

系统会自动创建一个包含92个RSS源的订阅。

### 2. 查看订阅详情

添加成功后，订阅列表会显示：
- **订阅名称**: AK推荐
- **信息源数量**: 92个RSS源
- **状态**: 启用/禁用
- **抓取频率**: 你设置的频率

### 3. 批量抓取内容

点击"立即抓取"按钮，系统会：
1. 并发访问所有92个博客的RSS源
2. 抓取最新文章（通常每个博客10-30篇）
3. 自动去重（基于URL和标题）
4. 保存到本地IndexedDB
5. 显示成功消息（新增内容数量）

### 4. 编辑订阅（高级功能）

点击"编辑"按钮可以：
- 查看所有92个博客源的列表
- 单独启用/禁用某个博客源
- 添加/删除关键词过滤
- 修改订阅名称

## 技术实现

### 数据结构

```typescript
// AK推荐预设定义
{
  name: 'AK推荐',
  rss: 'BUNDLE:HN_POPULAR_BLOGS', // 特殊标识
  keywords: ['tech', 'blog', 'programming', 'ai', 'startup']
}
```

### 订阅创建逻辑

当用户选择"AK推荐"时：

```typescript
if (isBundleSubscription) {
  const bundleSources = HN_POPULAR_BLOGS.map(blog => ({
    type: 'rss' as const,
    url: blog.rss,
    enabled: true
  }))

  await subscriptionService.createSubscription({
    name: 'AK推荐',
    company: 'AK推荐',
    sources: bundleSources, // 92个RSS源
    keywords: ['tech', 'blog', 'programming', 'ai', 'startup'],
    enabled: true,
    frequency: userSelectedFrequency
  })
}
```

## 性能考虑

### 首次抓取
- **RSS源数量**: 92个
- **预计时间**: 30-60秒（取决于网络速度和服务器响应）
- **预计新增内容**: 500-1500篇文章
- **存储空间**: 约5-10MB（IndexedDB）

### 后续抓取
- **增量更新**: 只抓取新文章
- **去重机制**: 自动跳过已存在的内容
- **平均新增**: 50-200篇/天（取决于博客更新频率）

### 优化建议
1. **首次使用建议频率**: 每天（避免频繁抓取）
2. **观察一周后**: 可调整为每周（减少冗余）
3. **关键词过滤**: 添加感兴趣的关键词，减少无关内容

## 常见问题

### Q1: 还能单独订阅某个HN博客吗？
A: 可以！使用"自定义添加"功能，输入博客名称和RSS地址即可。但推荐直接使用AK推荐，更方便管理。

### Q2: 为什么下拉菜单中不显示92个博客了？
A: 为了简化界面，避免下拉菜单过长。所有92个博客已打包在"AK推荐"中，一键订阅即可。

### Q3: 添加AK推荐后还想单独订阅某个其他博客怎么办？
A: 使用"自定义添加"功能，输入博客名称和RSS地址即可创建独立订阅。

### Q4: AK推荐的博客源可以删除某些吗？
A: 可以！编辑订阅时，单独禁用不需要的博客源即可。

### Q5: 抓取92个博客会不会太慢？
A: 系统使用并发抓取，通常30-60秒完成。首次可能较慢，后续增量更新很快。

### Q6: 内容太多怎么办？
A: 两种方法：
1. 添加关键词过滤（如 "AI", "React", "Rust"）
2. 在编辑订阅时禁用部分博客源

### Q7: 可以修改抓取频率吗？
A: 可以！编辑订阅即可修改频率（每小时/每天/每周）。
